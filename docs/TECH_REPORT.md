# Технический отчет: URL Shortener API

## Цель
- Сделать второй сильный backend-проект для резюме.
- Показать работу с API, БД, кэшированием, rate limiting и тестами.

## Что реализовано
- Сокращение ссылок с кастомным alias.
- Редирект и подсчет кликов.
- TTL-ссылки (статус `410` после истечения).
- Redis-кэш резолва ссылок.
- Ограничение запросов по IP в скользящем окне.

## Архитектура
- `app/api` - endpoint слой.
- `app/services` - бизнес-логика: shortener/cache/rate limit.
- `app/models` - модель `Link`.
- `app/db` - SQLAlchemy-сессии.
- `alembic` - миграции схемы.

## Интересные решения
- Кэш через Redis, но с безопасным fallback (если Redis недоступен, API продолжает работать через БД).
- Rate limiting реализован как отдельный сервис, подключаемый в endpoint-слое.

## Ошибки и исправления
- Будет дополняться по ходу разработки и тестирования.

## Тестирование
- Интеграционные тесты:
  - создание ссылки;
  - редирект + инкремент счетчика;
  - коллизия alias.
- Проверки: `ruff`, `pytest`.
